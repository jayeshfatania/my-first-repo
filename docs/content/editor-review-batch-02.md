## 1. BATCH OVERVIEW

Batch 02 is a stronger first pass than Batch 01. The persona voices are more confident throughout, specific details are sharper, and a handful of descriptions are genuinely excellent (symonds-yat, newtondale-levisham, kinnoull-hill). No full rewrites are required — all issues are single-sentence fixes or targeted edits.

Four systemic problems carry forward from Batch 01 or emerge fresh here:

**1. Walk-name openers** — four cases again (beacon-fell, bradgate-park, calke-abbey, elterwater-langdale). The copywriter has not embedded the "no walk name in sentence one" rule as automatic behaviour. This needs to be a pre-submission checklist item, not a post-hoc edit.

**2. Deborah's "Ideal for" closer** — appears in three of her four descriptions this batch (dalby-forest, moors-valley, elterwater-langdale). "Ideal for families", "Ideal for a proper half-day out", "Ideal for a half-day outing" — three variants of the same formula. As with "which is always a bonus" in Batch 01, this has tipped from a character trait into a verbal tic. Two of the three must change.

**3. Morag's café/pub omission** — the persona spec is explicit: "Will always mention if there is a good pub or café nearby." Four of five Morag descriptions in this batch omit it (roseberry-topping, kinnoull-hill, loch-ard-aberfoyle, rhossili-down). The fix in all cases is adding a half-sentence to an existing closer — no structural changes needed.

**4. One cross-persona imperfection error** — lyme-park applies Jamie's "there/their" confusion to a Pete description. Pete's imperfection set is dropped apostrophes only. Fix is one word.

Sixteen descriptions need a single targeted fix. Four pass without changes.

## 3. PATTERNS AND SYSTEMIC ISSUES

---

### Issue 1: Walk-name openers
**Frequency:** 4 of 20 descriptions (beacon-fell, bradgate-park, calke-abbey, elterwater-langdale)
**Pattern by persona:** Pete ×2, Priya ×1, Deborah ×1. Jamie and Morag are clean on this.
**Observation:** The pattern from Batch 01 is repeating. This is a pre-submission check the Copywriter needs to run on every entry before delivery. Suggest adding a one-line checklist to the brief: "Does sentence 1 open with or prominently contain the walk ID name or location name? If yes, rewrite."

---

### Issue 2: Deborah's "Ideal for" closer
**Tally this batch:** dalby-forest ("Ideal for families…"), moors-valley ("Ideal for a proper half-day out"), elterwater-langdale ("Ideal for a half-day outing with the whole family")
**Assessment:** "Ideal for" has become Deborah's default closer in the same way "which is always a bonus" was in Batch 01. It appears three times in four descriptions. Two of the three are flagged above for replacement. After fixes, one instance may remain — dalby-forest's replacement closer is suggested above; moors-valley and elterwater-langdale replacements are provided. Validator should confirm only one "Ideal for" survives across the full Deborah set.

---

### Issue 3: Morag café/pub omission
**Tally this batch:** roseberry-topping, kinnoull-hill, loch-ard-aberfoyle, rhossili-down — all four missing.
**Total Morag entries missing café/pub:** check against Batch 01 (malham-cove and tentsmuir-forest-fife both absent — 2 instances). Combined: 6 Morag entries across two batches lack this. The persona spec says "Will always mention if there is a good pub or café nearby" — this is being treated as optional rather than mandatory.
**Recommended action:** Add to Copywriter brief: "Morag must always close with or include a pub or café mention. If the immediate area has nothing, note the nearest town option." All four batch-02 fixes are provided above — half-sentence additions to existing closers.

---

### Issue 4: Cross-persona imperfection error
**Entry:** lyme-park (Pete) — "there website" applies Jamie's there/their confusion to a Pete description.
**Fix:** One word. Either "its website" (Pete's apostrophe-drop pattern applied correctly) or "their website" (no imperfection, clean).
**Wider note:** Persona imperfections need to be persona-specific. The copywriter appears to have a pool of "authentic errors" being distributed without checking which persona they belong to.

---

### Issue 5: Priya opener patterns
"We stumbled [on/across] this one" now appears in frensham-common (Batch 01) and penllergare-valley-woods (Batch 02). The construction is identical even if the words differ slightly. Priya has several opener options: subverted expectations (lickey-hills-birmingham, Batch 01), time/condition specific (grasmere-lake, Batch 01), and discovery (frensham-common). The discovery pattern should not be reused.
